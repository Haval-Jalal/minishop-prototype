# AI Usage Log

## Prompt 1
"Generate a simple UML class diagram for a mini e-commerce domain including products and cart."

AI Output:
The AI suggested several classes including Product, Cart, Order, Payment, and User.

Manual Changes:
Order and Payment were removed to reduce scope and better align with assignment requirements. The model was simplified to focus on Product, Cart, CartItem, Category, and User.

---

## Prompt 2
"Create a minimal JavaScript function for adding items to a shopping cart using localStorage."

AI Output:
The AI generated a longer script including array handling and multiple helper functions.

Manual Changes:
The script was simplified to a basic counter-based solution to match the prototype level and keep the implementation under 50 lines.

---

## Prompt 3
"Provide a clean and minimal CSS layout for a product list."

AI Output:
The AI generated a grid-based layout with several advanced styling features.

Manual Changes:
The styling was simplified to ensure consistency, maintainability, and clarity while keeping a professional structure.

---

## Reflection

AI was useful for generating structure and initial ideas, but the outputs required simplification and refinement to fit the project scope. I learned that AI works best as a support tool, not as a replacement for understanding system design and requirements.
